# PRINT SPEC — the product contract
*Chris, 2026-07-02: "Give the system the palette, fonts, ICP, and advisor details — and it prints an
incredibly amazing website that looks like a $25–50K team built it." This doc is that contract.*

## LOCKED (invariant across every site — never regenerated, never improvised)

1. **The structure.** The conversion skeleton: Hero → Pain → ICP self-sort → Fit/Capture (the MQL
   gate) → Services → Process → Proof → Final CTA → Insights/Content (incl. video) → Footer.
   (`docs/REFERENCES.md` RULE #1 + the CRO rules in RULE #2. Agent 05 blocks ship on violations.)
2. **Component quality.** Every visual element comes from the blessed library
   (`templates/components/catalog.html` + `docs/harvest/`) — measured from the reference sites,
   gate-tested, Chris-approved by ID. Builders ASSEMBLE; they never re-derive. New components enter
   only through the harvest→curate→Chris loop.
3. **Copy quality.** RMBC-grounded, ICP-resonant: exact-words pain language, named mechanism,
   insider fluency, kill-list enforced, zero fabricated claims (`[VERIFY]` discipline; compliance
   overlay per the client's regulatory regime).
4. **The QA gauntlet.** Deterministic gate (0 P0) + taste-critic pass + tier-appropriate external
   verification (LEDGER: no builder's self-report is ship evidence). AA contrast with alpha-blend math.

## FLEXIBLE (the client inputs — this is ALL a client needs to provide)

- **Brand:** palette (hex set) + fonts (or "pick for register")
- **ICP:** who, their exact pains, insider language (or research docs to derive it)
- **Advisor/client details:** name, story, credentials, real numbers, real assets (photos/video)
- Optional: register adjectives ("warm/institutional/confident/technical") to steer the director

## THE PRINT PIPELINE (inputs → site)

1. **Intake** → the four inputs above (client-fillable, COPY_DOC_FOR_ALEX.md is the model).
2. **Research/Copy** → RMBC agents produce ICP brief + copy keyed to the skeleton slots.
3. **Director** → per `templates/ASSEMBLY.md`: pick type register + tonal map from adjectives;
   draw the composition vector from the library (adjacency law + 40% same-vertical distance rule);
   imagery/motion dials from the client's REAL assets; log vector to the build registry.
4. **Build** → tier-appropriate model assembles components + retints tokens + drops copy.
5. **Gauntlet** → gate + taste critic + repair loop until clean.
6. **Output** → single-file pages + review link. Human gate before deploy.

## THE $25–50K "TEAM-BUILT" SIGNALS (what separates printed-by-us from generated-by-AI)

Positive tells the system must hit (the inverse of the anti-slop list):
1. **A bespoke signature moment** that literalizes the client's mechanism (the convergence diagram
   class) — something no template has.
2. **Real deliverable/artifact shown** (C-26 class) — the plan document, the actual output, not stock.
3. **Motion with intent:** staggered reveals, drawn diagrams, count-ups with guards, one signature
   motion — never ambient decoration, never particles.
4. **Editorial detail:** measured type scales, own-line accent phrases, hairline systems, numeral
   treatments, real rag/measure discipline.
5. **Photography/video direction** consistent with §6 imagery rules — warm-documentary, the client's
   actual world; video as first-class content (click-to-play stories).
6. **Copy specificity:** insider terms used fluently, numbers with units, named ICP in the first
   viewport — reads like someone who knows the client's world wrote it.
7. **Compliance polish:** disclosures present and correct — the tell that a real firm shipped it.
8. **Zero slop tells:** no gradient text, no icon-chip heroes, no uniform card grids, no
   bounce easing, no #000, no shadowed-white-card defaults, no Inter-as-display.
