# PALETTES — the curated palette library (a selector, not a generator)

**The law this file exists to enforce:** the Director NEVER invents hex values. It SELECTS a
blessed palette from this library (routing by the client's register adjectives + vertical),
then at most RETINTS within the entry's solved math (e.g., swap the accent toward a client's
existing logo color, re-solving the AA rules). "What it was generating before was not great"
(Chris, 2026-07-08) — because it was generating. Now it picks.

**Every entry ships complete:** full token set, dark-band recipe, accent-budget notes, AA
contrast pre-solved, register tags, and a provenance line (where it was proven). New entries
enter ONLY via harvest→curate→Chris blessing — same as patterns.

**Status meanings:** BLESSED (Chris approved, selectable) · SEED (well-founded, awaiting
Chris's bless/kill pass) · CANDIDATE (harvested idea, not yet solved).

## Selection procedure (Director, stage 2.5)

1. Client has a brand guide → their tokens win; this library only fills gaps (dark-band recipe,
   accent budget) and the AA math still gets solved.
2. No brand guide → route adjectives/vertical to the 1–2 nearest entries, state the pick + one
   sentence of rationale in the DESIGN_BRIEF. Chris sees the palette NAME in the brief — a
   1-second check ("Indigo Dawn again? no — Ledger for this one").
3. Retint rule: one hue swap max (usually accent toward client logo), re-run every AA pair,
   note the deltas in the brief. Two same-vertical clients may not ship the same entry
   untinted (registry check).
