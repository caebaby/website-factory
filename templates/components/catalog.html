The Component Library — react by ID.

Every entry below is a worked example — an exhibit, not a template. Each carries three layers: INVARIANT (the measured craft numbers, always kept) · VARY (the mandatory freshness axes every new build must move on) · NEVER PORT (instance artifacts born from one client's mechanism — re-derived from THIS client's research, every time). Executors receive the INVARIANT/VARY/NEVER-PORT block from the Director's brief, not this raw HTML — and a verbatim rebuild of any exhibit is a critic-flagged defect (stamp detection). The toggle (top right) re-skins everything live: retint the colors, keep the numbers. React by ID — keep, kill, or change.
