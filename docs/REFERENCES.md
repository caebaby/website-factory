# Website Factory — Design & Conversion Reference Library

> This document is the calibration standard for build agents. Read it before generating any site. Every decision — typography, color, layout, copy — should be traceable to a principle here. If you can't trace it, cut it.

---

## 1. The Design Quality Standard

What separates premium from AI slop is not visual complexity. It is the presence of decisions that could only have been made by someone who had a specific opinion at a specific moment. AI generates plausible averages. Premium sites are opinionated.

### Principle 1: Absence as Signal

The discipline of removal distinguishes premium work. Every element on a premium page is there because someone decided to keep it — not because it was generated by default. If an element cannot answer "why am I here?", it should not be on the page.

**Specific techniques:**
- Remove decorative icons. Use none unless each carries distinct semantic meaning.
- Remove gradient backgrounds under text. Gradients belong behind product art only, never behind content.
- Remove stock photography of humans unless the human is the actual ICP being targeted.
- Remove hero subheadlines that merely restate the headline in different words.
- Remove section dividers (borders, rules). Use whitespace alone to separate sections.

**Test:** Could you remove this element and the page would communicate its point more clearly? If yes, remove it.

---

### Principle 2: Light Weight as Luxury Signal

Bold headlines at large sizes read as insecure. Stripe's entire premium register comes from weight-300 display headlines at 72px+. At large sizes, light weight signals effortless confidence. Bold signals someone trying to sound authoritative.

**Specific techniques:**
- Display headlines: weight 200–300, never 700 for premium registers.
- Negative letter-spacing at large sizes: -0.02em to -0.04em at 64px+.
- Body text: weight 400, never bumped to 500 for "legibility" on desktop.
- Line height on headlines: 1.0–1.1. Not 1.4. Headlines are not paragraphs.
- Exception: dark-mode sites must bump weight one level (300 → 400, 400 → 500) because dark backgrounds optically thin letterforms.

**Test:** Does the headline feel like it's trying to convince you of its authority? If yes, reduce the weight.

---

### Principle 3: Single Accent, Maximum Impact

Multiple accent colors means no accent colors. One accent color, used exclusively for the most important interactive element on each viewport. When it appears, it should feel like a signal — because it only appears once.

**Specific techniques:**
- Choose one accent. Use it for primary CTAs only. Nothing else.
- If you need a secondary color, use a tint of the same hue at 30% opacity.
- Semantic accent use: Ramp's green appears only for financial wins. Stripe's violet appears only on primary CTAs. The color is trained to mean something.
- Never use the accent color in decorative patterns, borders, or illustrations.

**Test:** Count the number of times your accent color appears on the page. If it's more than 3, remove all but the 3 most important instances.

---

### Principle 4: Authored Interaction States

AI-generated UIs only build the default state. Every site produced by a non-human agent has missing hover states, generic focus rings, and broken disabled states. This is the most reliable tell. Premium sites author every element through its complete lifecycle.

**Required states for every interactive element:**
- Default
- Hover (distinct from default — not just opacity change)
- Focus (accessible, high-contrast ring, not browser default)
- Active / pressed (visual compression or color shift)
- Loading (spinner or skeleton, never stale content)
- Disabled (reduced opacity + cursor: not-allowed)
- Error (inline message, red border, never just an alert)

**Test:** Tab through the page with keyboard only. If any interactive element looks broken or invisible in focus state, it fails.

---

### Principle 5: Functional Motion Only

AI-generated sites animate everything because motion signals effort. Premium sites animate only things that carry information. Motion earns its place by communicating state, sequence, or causality — not by adding visual interest.

**Specific techniques:**
- Scroll-triggered entrance: stagger child elements entering in sequence (50ms offset per item, not all at once).
- State transitions: color and opacity changes on hover should be 150–200ms, not instant and not slow.
- No looping ambient animations on page load unless the animation is the product (like Lusion's 3D).
- Same easing curve (cubic-bezier) across all animations on the site. Not a mix of ease, ease-in, ease-out, and linear.
- Page transitions: if used, maintain spatial continuity. The exiting page should feel like it's going somewhere, not just fading.

**Test:** Watch the page load and the first 10 seconds of interaction. Identify every animation. For each: what information does this communicate? If the answer is "none," remove it.

---

### Principle 6: Structural Groundedness

Whitespace only feels confident when the underlying grid is exact. Inconsistent spacing — elements that are almost-but-not-quite aligned — reads as carelessness. The Blueprint Grid technique makes invisible structure visible.

**Specific techniques:**
- 4px or 8px base grid. Never mix units. Every spacing value is a multiple of the base.
- Blueprint Grid overlay on hero: 1px grid lines at 10–15% opacity (`rgba(255,255,255,0.1)` on dark, `rgba(0,0,0,0.06)` on light). Felt before consciously seen.
- Border-radius consistency: pick one value (4px, 8px, or 12px) and apply it to everything. Never have rounded cards next to sharp-cornered buttons.
- No element should be "eyeballed" into position. If it can't be expressed in grid units, reconsider whether it belongs.

**Test:** Overlay a grid on the page. Every element should snap to it within 2px. If 20% of elements don't, the spacing system is broken.

---

### Principle 7: Specificity as Credibility

Vague claims produce skepticism. Specific claims produce belief because they imply that someone measured something. Every proof claim must contain a number, a name, a timeline, or a process step. "We help advisors grow" is worthless. "Advisors who join retain an average of 94% of their book in year one" is credible.

**Specific techniques:**
- Replace every adjective (successful, proven, comprehensive) with evidence.
- Use named processes instead of vague process language: "We review your book before the first call" instead of "we do thorough preparation."
- If no hard metric exists, use specificity of situation: "Advisors from Merrill, Raymond James, and Edward Jones" is more credible than "advisors from major wirehouses."
- Numbers in headlines outperform all other headline formats for B2B conversion.

**Test:** Read every claim on the page. For each: what's the evidence? If the answer is "trust us," rewrite it.

## 3. Typography Patterns

### Pattern 1: The Stripe Stack (Light-Weight Display)
- Display heading: Variable or static sans-serif at weight 200–300, size 64–96px, tracking -0.03em, line-height 1.05
- Section heading: Same family, weight 300–400, size 36–48px, tracking -0.02em, line-height 1.1
- Body: Weight 400, size 16–18px, tracking 0, line-height 1.6
- Labels / eyebrow: All-caps, weight 500–600, size 11–12px, tracking 0.12–0.15em
- **Use for:** Light-background premium sites targeting sophisticated buyers

### Pattern 2: The Linear Stack (Calibrated Dark)
- Display heading: Inter Display, weight 400 (bumped from 300 on light), size 64–80px, tracking -0.02em, line-height 1.05
- Body: Inter Regular, weight 400, size 16px, tracking 0.01em (added for dark legibility), line-height 1.65
- Code/mono: Geist Mono or JetBrains Mono at 14px
- Background: #0F0F0F, headline color #FFFFFF, body color rgba(255,255,255,0.7)
- **Use for:** Dark-mode sites targeting technical or software-adjacent buyers

### Pattern 3: The Editorial Counterweight Stack
- Display: Tight-tracked, heavy, high-contrast serif or editorial sans (DM Serif, Playfair, or Canela) at weight 700, size 80–120px, tracking -0.04em, line-height 0.95–1.0
- Body: Clean geometric sans at weight 300–400, size 16–18px, line-height 1.7
- **Use for:** Hero sections with photography or video — the type creates stillness against the movement

### Pattern 4: The Technical Authority Stack (Vercel/Ramp)
- Display: Geometric sans (Geist, Inter, or DM Sans) at weight 500–600, size 48–64px, tracking -0.01em, line-height 1.1
- Body: Same family at weight 400, size 17px, line-height 1.65
- Mono accent: Geist Mono or Berkeley Mono for code or data snippets — signals precision
- **Use for:** Data-forward, metrics-heavy sections. The mono creates technical credibility.

### Critical Typography Rules (Never Violate)
- Never mix more than 2 font families on one page.
- Never use a web-safe fallback font (Arial, Georgia, Times) as the primary face — always load a proper webfont.
- Inter is acceptable as a body font; it is insufficient as a display font without optical sizing adjustments.
- "Inter Display" is a different font than "Inter." Use the correct variant.
- Never set line-height below 1.4 for body text.
- Never set line-height above 1.15 for display headings.
- Never track body text wider than 0.02em.

## 4. Color & Depth Techniques

### Shadows

**The Stripe Shadow (card depth):**
```css
box-shadow:
  0 30px 60px -12px rgba(50, 50, 93, 0.25),
  0 18px 36px -18px rgba(0, 0, 0, 0.3);
```
Use on: feature cards, pricing cards, product screenshots. Never on flat decorative containers.

**The Subtle Lift (interactive elements):**
```css
box-shadow:
  0 4px 6px -1px rgba(0, 0, 0, 0.1),
  0 2px 4px -1px rgba(0, 0, 0, 0.06);
```
Use on: buttons on hover, form inputs on focus.

**Never use:**
- `box-shadow: 2px 2px 5px #000` — this is 1995
- Colored drop shadows on colored backgrounds without intentional color theory
- Identical shadow on every element on the page — differentiate by elevation level

---

### Backgrounds

**Light sites:**
- Primary background: #FFFFFF or #FAFAFA (never pure gray like #F5F5F5 which reads as dated)
- Secondary sections: #F8F9FA or #F4F5F6 — 2–3% off-white, not gray
- Never use card borders when a subtle shadow + slightly different background can separate content

**Dark sites:**
- Base: #0F0F0F or #111111. Never #000000.
- Card surface: #1A1A1A or #181818
- Elevated surface: #222222 or #242424
- Never stack more than 3 distinct dark surface levels — it gets visually confusing

**Blueprint Grid (for hero sections):**
```css
.hero {
  background-image:
    linear-gradient(rgba(255,255,255,0.07) 1px, transparent 1px),
    linear-gradient(90deg, rgba(255,255,255,0.07) 1px, transparent 1px);
  background-size: 48px 48px;
}
```
Light version: replace `rgba(255,255,255,0.07)` with `rgba(0,0,0,0.05)`.

---

### Color Palettes

**Monochrome Plus One (Vercel/Ramp model):**
- Base: Black and white or near-black and off-white
- Accent: One color, one application. Examples:
  - Stripe: #533AFD (violet) — CTAs only
  - Ramp: #00B06E (green) — positive outcomes only
  - Mercury: #A78BFA (warm violet) — interactive elements only

**Financial Services Authority Palette:**
- Deep navy headline: #061B31 (Stripe's exact value — works)
- Body text: #374151 or #4B5563
- Muted text: #9CA3AF
- Background: #FFFFFF
- Accent: One deep, saturated color. Navy + gold, navy + forest green, charcoal + copper. Not navy + teal.

**Never:**
- Use gradients as section backgrounds under text
- Use #003087 "financial services blue" — it screams generic
- Mix warm and cool tones without intentional contrast
- Use green for anything other than positive financial outcomes — it's a semantic color

---

### Texture and Depth Without Decoration

- Use `backdrop-filter: blur(16px) saturate(180%)` for glassmorphism elements — but only on top of photographic content, not flat color backgrounds
- Use `mix-blend-mode: multiply` sparingly to integrate brand color with photography
- Use noise texture at 2–4% opacity on hero backgrounds to prevent "digital flatness" — a subtle grain reads as premium, not cheap
- Avoid CSS gradient mesh generators — they produce obvious AI outputs. If using a gradient, hand-specify the stops.

## 5. Layout Patterns

### The Museum Grid (Pentagram/Portfolio approach)
Items presented in a consistent-proportion grid with equal padding on all sides. No cards, no borders — just work on a white surface with breathing room.
```
[Image]   [Image]   [Image]
Title     Title     Title
Category  Category  Category
```
Use for: case studies, outcomes, advisor profiles. Not testimonials.

### The Problem-Solution Column (Mercury/Ramp approach)
Left column: problem statement in large type. Right column: product screenshot or data visualization. Alternates sides as you scroll. Simple. Not a "features" grid.
```
"You don't own        [Product
your client           screenshot
relationships."       showing ownership]
```

### The Cinematic Hero (Brex approach)
Full-width, dark background, one number rendered at 80–120px in the center-left, brief context copy below, CTA below that. No imagery other than subtle product art at the right edge. Not a slider. Not a video.

### The Stacked Proof Section (Directive/Ramp approach)
Metric → Source → One-line context. Repeat 3–4 times in a horizontal or vertical rhythm.
```
94% book retention    Axiom advisor cohort, 2023–2025
$2.1M average AUM     At time of transition
90-day transition     Median, not outlier
```

### The Transparent Process Section (StoryBrand/Instapage approach)
Three numbered steps immediately below the CTA:
```
01.                  02.                  03.
You book 20 min.     We review your       You leave with
No prep required.    situation before     clarity — whether
                     we talk.             you join or not.
```
Text-only. No icons. The numbers do the work.

### The Self-Sort Hero (Lyft approach)
Split hero presenting two distinct paths for two distinct visitor states. Not a navigation menu — two CTAs that are the entire hero.
```
[I'm exploring what        |    [I've decided to move.
independence looks like]   |     Show me the Axiom model.]
```
Each path leads to a different page or section. Never a generic page.

### What Not to Build
- Testimonial carousel / slider — build a static grid of 3 specific outcomes instead
- Hero video backgrounds — almost always lazy. Specific product screenshots outperform.
- "Our process" timeline with icons — use numbered steps with copy, no icons
- Feature grid with 6 icons in two rows — signals a company that doesn't know its hierarchy of value
- Full-width email capture section in the middle of a page — it kills momentum

## 6. Component Patterns

### CTA Button
```css
/* Primary — the only place the accent color lives */
.btn-primary {
  background: var(--accent);
  color: white;
  padding: 14px 28px;
  border-radius: 8px;  /* match site border-radius system */
  font-weight: 500;
  font-size: 16px;
  letter-spacing: -0.01em;
  transition: all 150ms cubic-bezier(0.4, 0, 0.2, 1);
}
.btn-primary:hover {
  background: var(--accent-dark); /* 15% darker, not opacity */
  transform: translateY(-1px);
  box-shadow: 0 4px 12px rgba(var(--accent-rgb), 0.3);
}
.btn-primary:active {
  transform: translateY(0);
  box-shadow: none;
}

/* Secondary — ghost, never filled with a secondary color */
.btn-secondary {
  background: transparent;
  border: 1.5px solid rgba(0,0,0,0.2);
  color: var(--text-primary);
}
```

**Micro-copy under CTAs:**  
Always add 1 line below the CTA that handles the dominant objection at the moment of conversion.
- "Most advisors get clarity in one conversation. No pressure, no pipeline script."
- "No commitment required. Cancel anytime."
- "Takes 90 seconds. No spam."

---

### Navigation
- Max 4–5 items. Every item must be a destination, not a category.
- Sticky only if the page is long and the CTA needs to persist.
- No mega-menus for sites under 8 pages.
- Mobile: hamburger is acceptable. Drawer from right. Full-screen overlay is a 2015 trend.
- Active state: underline or left-border, not background highlight.

---

### Form Fields
```css
input, textarea, select {
  border: 1.5px solid #E5E7EB;
  border-radius: 6px;
  padding: 12px 16px;
  font-size: 16px; /* never smaller — prevents iOS zoom */
  transition: border-color 150ms ease;
}
input:focus {
  border-color: var(--accent);
  outline: none;
  box-shadow: 0 0 0 3px rgba(var(--accent-rgb), 0.15);
}
input:invalid:not(:placeholder-shown) {
  border-color: #EF4444;
}
```
- 5 or fewer fields. Every field above 5 reduces conversion.
- Label above field, not placeholder. Placeholders disappear on focus.
- Inline error messages, never alert boxes.

---

### Cards (Feature, Pricing, Case Study)
- Consistent border-radius across all cards on the site.
- Subtle background differentiation over visible borders: `background: #FAFAFA` on a white page reads cleaner than `border: 1px solid #E5E7EB`.
- If borders are used: 1px at 15–20% opacity, not full saturation.
- Hover state: lift via `translateY(-2px)` + shadow increase. Not background color change alone.
- Padding: minimum 24px on mobile, 32–40px on desktop. Never 12–16px — it reads as cramped.

---

### Eyebrow / Category Label
Small text above the main headline that signals what kind of thing this is.
```css
.eyebrow {
  font-size: 11px;
  font-weight: 600;
  letter-spacing: 0.12em;
  text-transform: uppercase;
  color: var(--accent);  /* or muted text — never body text color */
}
```
Use sparingly — one per section, not one per card. Overuse destroys the hierarchy effect.

---

### Testimonials
Wrong approach: carousel with star ratings and headshots.  
Right approach: a static section with 2–3 attributed quotes, name + title + company, no stars, in a clean two-column or three-column grid. The specific attribution is the credibility — not the star count.

If using a quote, the quote must be specific: not "Chris helped me grow my business" but "In year one with Axiom, I retained every client from my prior firm and added $3.2M in new AUM — while working fewer hours."

## 7. Anti-Slop Checklist

These are specific signals that identify AI-generated or low-quality work. Never produce any of these.

### Typography Slop
- [ ] Bold headlines at large sizes (700+ weight at 64px+) — weight-300 or weight-400 instead
- [ ] Tight line-height on body text (below 1.4) — reserve tight leading for display only
- [ ] Wide tracking on body text (above 0.02em) — tracking is for labels and eyebrows, not paragraphs
- [ ] Mixed font families (3+) — one sans, one optional mono or serif, nothing else
- [ ] Placeholder fonts (Arial, Georgia) anywhere visible — always load a proper webfont
- [ ] Same font weight used for headlines and body — differentiate by at least 200 weight units
- [ ] Line-height above 1.15 on display headlines — tighten it

### Color Slop
- [ ] Multiple accent colors (more than one brand color used for interactive elements)
- [ ] Gradient backgrounds under text — gradients belong behind product art only
- [ ] Pure #000000 black background — use #0F0F0F minimum
- [ ] Stock blue (#003087, #0066CC, #1E40AF) as primary brand color — it screams generic fintech
- [ ] Muddy tertiary colors — every color in the palette should be derivable from 2–3 hue decisions
- [ ] Green used for anything other than positive financial outcomes (savings, growth)
- [ ] Accent color appearing more than 3 times per viewport

### Layout Slop
- [ ] Testimonial carousel / slider — replace with static grid
- [ ] Hero video background — replace with product screenshot or still image
- [ ] 6-icon feature grid in two rows — cut to 3, or replace with outcome statements
- [ ] Decorative dividers (horizontal rules between sections) — use whitespace
- [ ] Spacing that doesn't snap to a grid — audit with a base-8 overlay
- [ ] Border-radius inconsistency (some elements 4px, some 16px, some fully rounded)
- [ ] Content that requires horizontal scrolling on mobile

### Copy Slop
- [ ] Vague benefit claims ("we help advisors grow") — always quantify or name-drop
- [ ] Headline that names the company or product before naming the problem — problem first
- [ ] "We" statements where "you" statements belong — the customer is the hero
- [ ] Generic CTAs: "Learn More," "Get Started," "Contact Us" — name the exact next action
- [ ] Industry jargon in headlines ("fee-only fiduciary independent RIA") — use plain language first
- [ ] Testimonials without specific outcomes — "Jane helped me" is not a testimonial
- [ ] FAQ at the top of the page — FAQs go immediately before the final CTA
- [ ] No micro-copy under CTA buttons — always handle the dominant objection inline

### Interaction Slop
- [ ] Missing hover states on any clickable element
- [ ] Browser-default focus rings (blue outline on :focus) — custom high-contrast focus style always
- [ ] Missing disabled state styling on form submit buttons
- [ ] No loading state on form submission — always show spinner or confirmation
- [ ] All elements animating simultaneously on page load — stagger by 50ms
- [ ] Mixed animation easing (some ease-in, some ease-out, some linear) — pick one curve
- [ ] Animations that loop continuously without user interaction
- [ ] Scroll-jacking that overrides native scroll behavior

### Photography / Visual Slop
- [ ] Stock photos of smiling businesspeople shaking hands — never
- [ ] Abstract geometric illustrations as hero visual — use product screenshots or real outcomes
- [ ] Icon libraries (Font Awesome, Heroicons) used decoratively — remove or replace with purpose-built icons
- [ ] Illustrations that don't match the visual tone of the typography — mismatched aesthetic registers
- [ ] Gradients that look randomly generated — if using a gradient, hand-specify every stop
- [ ] Product screenshots with placeholder data — always show real or realistic data

### Financial Services Specific Slop
- [ ] Leading the hero with credentials ("20 years of experience") instead of the client's problem
- [ ] Listing services as bullet points ("Financial Planning · Investment Management · Tax Strategy") instead of showing outcomes
- [ ] Vague ownership language ("independent," "fee-only") without explaining what it means to the client
- [ ] Compliance-softened copy ("may help," "potentially," "subject to") dominating above the fold — push compliance language to disclaimers
- [ ] No self-sort / segmentation — one page trying to speak to both "exploring" and "decided" visitors
- [ ] Missing transparent process section — not telling the visitor what happens after they click
