# Website Factory — Design & Conversion Reference Library

> This document is the calibration standard for all build agents.
> Read it before generating any site. Every layout, copy pattern, and interaction decision should be traceable to a principle here.

---

## THE NON-NEGOTIABLE RULE — Read This First

**The conversion skeleton is constant. The archetype is the skin.**

Every site built in this factory — regardless of archetype, template, client, or aesthetic direction — hits these 9 sections in this exact order. Archetypes (Institution, Editor, Modernist, Heritage) change how each section looks and feels. They do not move, remove, or reorder any section.

| # | Section | Conversion Job |
|---|---------|---------------|
| 1 | **Hero** | Hook the right visitor in 3 seconds. Headline names the pain or outcome. Dual CTA: high-commitment + low-commitment. |
| 2 | **Pain Validation** | Make the ICP feel deeply understood before pitching anything. 3–4 specific pain points in the client's exact words. |
| 3 | **ICP Self-Sort** | Help right-fit visitors identify themselves. Help wrong-fit visitors self-exit. Never speak to everyone. |
| 4 | **Fit Assessment / MQL Capture** | The lead gate. 4 qualifying questions + email. Primary conversion event on the page. |
| 5 | **Services / Solution** | Show what you actually do, tied to the pains named above. Not a feature list — an outcome list. |
| 6 | **Process** | Remove fear of reaching out. Show exactly what happens after they click. Reduces no-shows and cold feet. |
| 7 | **Proof** | Specific outcomes. Numbers, names, timelines. Never vague social proof. |
| 8 | **Final CTA** | Dual path close: high-commitment (book a call) + low-commitment (assessment). Last chance to convert. |
| 9 | **Footer** | Navigation, contact, compliance disclaimer. Required before any site goes live. |

**This sequence is not a suggestion. Agent 05 will fail QA on any site missing a section or breaking the order.**

The only flexibility: optional sections (blog preview, video, content hub) may be inserted between sections 5–7 when the client has content assets. They cannot replace or displace the 9 core sections.

## 1. The Design Quality Standard

What separates premium from AI slop is not visual complexity. It is the presence of decisions that could only have been made by someone who had a specific opinion at a specific moment. AI generates plausible averages. Premium sites are opinionated.

### Principle 1: Absence as Signal

The discipline of removal distinguishes premium work. Every element on a premium page is there because someone decided to keep it — not because it was generated by default.

**Specific techniques:**
- Remove decorative icons. Use none unless each carries distinct semantic meaning.
- Remove gradient backgrounds under text. Gradients belong behind product art only.
- Remove stock photography of humans unless the human is the actual ICP being targeted.
- Remove hero subheadlines that merely restate the headline in different words.
- Remove section dividers (borders, rules). Use whitespace alone to separate sections.

**Test:** Could you remove this element and the page would communicate its point more clearly? If yes, remove it.

---

### Principle 2: Light Weight as Luxury Signal

Bold headlines at large sizes read as insecure. Stripe's entire premium register comes from weight-300 display headlines at 72px+. At large sizes, light weight signals effortless confidence.

**Specific techniques:**
- Display headlines: weight 200–300, never 700 for premium registers.
- Negative letter-spacing at large sizes: -0.02em to -0.04em at 64px+.
- Body text: weight 400, never bumped to 500 for "legibility" on desktop.
- Line height on headlines: 1.0–1.1. Not 1.4. Headlines are not paragraphs.
- Exception: dark-mode must bump weight one level (300→400) because dark backgrounds optically thin letterforms.

**Test:** Does the headline feel like it's trying to convince you of its authority? If yes, reduce the weight.

---

### Principle 3: Single Accent, Maximum Impact

Multiple accent colors means no accent colors. One accent, used exclusively for the most important interactive element per viewport.

**Specific techniques:**
- Choose one accent. Use it for primary CTAs only. Nothing else.
- If you need a secondary color, use a tint of the same hue at 30% opacity.
- Semantic accent use: Ramp's green appears only for financial wins. Stripe's violet only on primary CTAs.
- Never use the accent in decorative patterns, borders, or illustrations.

**Test:** Count accent color appearances per page. If more than 3, remove all but the 3 most important.

---

### Principle 4: Authored Interaction States

AI-generated UIs only build the default state. Premium sites author every element through its complete lifecycle.

**Required states for every interactive element:**
- Default → Hover (distinct) → Focus (high-contrast ring) → Active (compression or shift) → Loading (spinner/skeleton) → Disabled (reduced opacity + not-allowed cursor) → Error (inline, never alert)

**Test:** Tab through the page with keyboard only. If any element looks broken in focus state, it fails.

---

### Principle 5: Functional Motion Only

AI sites animate everything because motion signals effort. Premium sites animate only things that carry information.

**Specific techniques:**
- Scroll-triggered entrance: stagger child elements 50ms per item, not all at once.
- State transitions: 150–200ms. Not instant, not slow.
- No looping ambient animations unless the animation is the product.
- One easing curve across all animations. Not a mix of ease, ease-in, ease-out, linear.

**Test:** For every animation: what information does this communicate? If the answer is "none," remove it.

---

### Principle 6: Structural Groundedness

Whitespace only feels confident when the underlying grid is exact. Inconsistent spacing reads as carelessness.

**Specific techniques:**
- 4px or 8px base grid. Never mix units.
- Blueprint Grid on hero: 1px lines at 10–15% opacity. Felt before consciously seen.
- Border-radius consistency: pick one value (4px, 8px, or 12px), apply everywhere.
- No element eyeballed into position. If it can't be expressed in grid units, reconsider it.

---

### Principle 7: Specificity as Credibility

Vague claims produce skepticism. Specific claims produce belief because they imply someone measured something.

**Specific techniques:**
- Replace every adjective (successful, proven, comprehensive) with evidence.
- Use named processes: "We review your book before the first call" not "we do thorough preparation."
- If no hard metric exists, use specificity of situation: "Advisors from Merrill, Raymond James, and Edward Jones" beats "advisors from major wirehouses."
- Numbers in headlines outperform all other B2B headline formats.

**Test:** Read every claim. For each: what's the evidence? If the answer is "trust us," rewrite it.

The assessment job: get the email. Qualify the lead. Make the CTA feel like clarity, not commitment.

---

*Category: Personal Brand / Author*

## 4. Typography Patterns

### Pattern 1: The Stripe Stack (Light-Weight Display)
- Display heading: weight 200–300, size 64–96px, tracking -0.03em, line-height 1.05
- Section heading: weight 300–400, size 36–48px, tracking -0.02em, line-height 1.1
- Body: weight 400, size 16–18px, tracking 0, line-height 1.6
- Labels / eyebrow: all-caps, weight 500–600, size 11–12px, tracking 0.12–0.15em
- **Use for:** Light-background premium sites targeting sophisticated professional buyers

### Pattern 2: The Linear Stack (Calibrated Dark)
- Display heading: Inter Display, weight 400, size 64–80px, tracking -0.02em, line-height 1.05
- Body: weight 400, size 16px, tracking 0.01em (added for dark legibility), line-height 1.65
- Background: #0F0F0F, headline: #FFFFFF, body: rgba(255,255,255,0.7)
- **Use for:** Dark-mode sites targeting technically sophisticated or financially sophisticated buyers

### Pattern 3: The Editorial Counterweight Stack
- Display: Tight-tracked serif or editorial sans (EB Garamond, DM Serif, Canela) weight 400–600, size 80–120px, tracking -0.04em, line-height 0.95–1.0
- Body: Clean geometric sans weight 300–400, size 16–18px, line-height 1.7
- **Use for:** Hero sections with photography or video — tight type creates stillness against movement

### Pattern 4: The Technical Authority Stack
- Display: Geometric sans (Geist, DM Sans) weight 500–600, size 48–64px, tracking -0.01em, line-height 1.1
- Body: same family weight 400, size 17px, line-height 1.65
- Mono accent: Geist Mono or Berkeley Mono for data snippets — signals precision
- **Use for:** Data-forward, metrics-heavy sections. Mono creates technical credibility.

### Critical Typography Rules
- Never mix more than 2 font families on one page.
- Inter is acceptable as body; it is insufficient as a display font without optical sizing adjustments.
- "Inter Display" is a different font than "Inter." Use the correct variant.
- Never set line-height below 1.4 for body text.
- Never set line-height above 1.15 for display headings.

## 5. Color & Depth Techniques

### Shadow System

**The Stripe Shadow (card depth):**
```css
box-shadow:
  0 30px 60px -12px rgba(50, 50, 93, 0.25),
  0 18px 36px -18px rgba(0, 0, 0, 0.3);
```

**The Subtle Lift (interactive hover):**
```css
box-shadow:
  0 4px 6px -1px rgba(0, 0, 0, 0.1),
  0 2px 4px -1px rgba(0, 0, 0, 0.06);
```

**Never use:** Single flat box-shadow. `2px 2px 5px #000`. Identical shadow elevation on every element.

---

### Backgrounds

**Light:**
- Primary: #FFFFFF or #FAFAFA (never pure gray #F5F5F5 — reads dated)
- Secondary sections: #F8F9FA — 2–3% off-white

**Dark:**
- Base: #0F0F0F or #111111. Never #000000.
- Card surface: #1A1A1A
- Elevated: #222222

**Blueprint Grid (hero sections):**
```css
.hero {
  background-image:
    linear-gradient(rgba(255,255,255,0.07) 1px, transparent 1px),
    linear-gradient(90deg, rgba(255,255,255,0.07) 1px, transparent 1px);
  background-size: 48px 48px;
}
```

---

### Color Palettes

**Monochrome Plus One (Vercel/Ramp):**
- Base: near-black and off-white
- One accent, one application. Ramp: #00B06E for positive outcomes only.

**Financial Services Authority Palette:**
- Deep navy headline: #061B31 (Stripe's exact value — works)
- Body: #374151 or #4B5563
- Muted: #9CA3AF
- Background: #FFFFFF
- Accent: navy + gold, navy + forest green, or charcoal + copper. Never navy + teal.

**Never:**
- Gradients as section backgrounds under text
- #003087 "financial services blue" — screams generic
- Green for anything other than positive financial outcomes

## 6. Layout Patterns

### The Problem-Solution Column (Mercury/Ramp)
Left column: problem in large type. Right column: product screenshot or data visualization. Alternates sides as you scroll. Clean. Not a features grid.

### The Cinematic Hero (Brex)
Full-width dark background, one number at 80–120px center-left, context copy below, CTA below that. Not a slider. Not a video.

### The Stacked Proof Section
Metric → Source → One-line context. Repeat 3–4 times.
```
94% book retention    Axiom advisor cohort, 2023–2025
$2.1M average AUM     At time of transition
90-day transition     Median, not outlier
```

### The Transparent Process Section
Three numbered steps immediately below the CTA:
```
01.                  02.                  03.
You book 20 min.     We review your       You leave with
No prep required.    situation before     clarity — whether
                     we talk.             you join or not.
```
Text-only. No icons. The numbers do the work.

### The Self-Sort Hero
Two distinct CTAs that are the entire hero — not a navigation menu:
```
[I'm exploring what        |    [I've decided to move.
independence looks like]   |     Show me the Axiom model.]
```

### What Not to Build
- Testimonial carousel / slider — static grid of 3 specific outcomes instead
- Hero video backgrounds — specific product screenshots outperform
- "Our process" timeline with icons — numbered steps with copy, no icons
- Feature grid with 6 icons in two rows — signals no hierarchy of value
- Full-width email capture mid-page — kills momentum

## 7. Component Patterns

### CTA Button
```css
.btn-primary {
  background: var(--accent);
  color: white;
  padding: 14px 28px;
  border-radius: 8px;
  font-weight: 500;
  font-size: 16px;
  letter-spacing: -0.01em;
  transition: background 150ms cubic-bezier(0.4,0,0.2,1),
              transform 150ms cubic-bezier(0.4,0,0.2,1),
              box-shadow 150ms cubic-bezier(0.4,0,0.2,1);
}
.btn-primary:hover {
  background: var(--accent-dark);
  transform: translateY(-1px);
  box-shadow: 0 4px 12px rgba(var(--accent-rgb), 0.3);
}
.btn-primary:active { transform: translateY(0); box-shadow: none; }
```

**Micro-copy under CTAs:** Always add 1 line handling the dominant objection.
- "30 minutes. No pitch. We'll tell you honestly if there's a fit."
- "Takes 90 seconds. No spam."

---

### Navigation
- Max 5 items. Every item a destination, not a category.
- Sticky only if the CTA needs to persist on a long page.
- Mobile: hamburger acceptable. Full-screen overlay is a 2015 trend — use a right-side drawer.
- Active state: underline or left-border, not background highlight.

---

### Form Fields
```css
input, textarea {
  border: 1.5px solid #E5E7EB;
  border-radius: 6px;
  padding: 12px 16px;
  font-size: 16px; /* never smaller — prevents iOS zoom */
  transition: border-color 150ms ease;
}
input:focus {
  border-color: var(--accent);
  outline: none;
  box-shadow: 0 0 0 3px rgba(var(--accent-rgb), 0.15);
}
```
- 5 or fewer fields. Every field above 5 reduces conversion.
- Label above field, never placeholder-only.
- Inline error messages, never alert boxes.

---

### Cards
- Consistent border-radius across all cards.
- Subtle background over visible borders: `background: #FAFAFA` reads cleaner than a 1px border.
- Hover: `translateY(-2px)` + shadow increase. Not background color change alone.
- Padding: 32–40px desktop, 24px mobile. Never 12–16px.

---

### Testimonials / Proof Cards
Wrong: carousel with star ratings and headshots.
Right: static grid, 2–3 quotes, name + title + company, no stars.

The quote must be specific: not "Chris helped me grow my business" — "In year one with Axiom, I retained every client from my prior firm and added $3.2M in new AUM while working fewer hours."

## 8. Anti-Slop Checklist

### Typography
- [ ] Bold headlines at large sizes (700+ at 64px+) — use weight 300–400 instead
- [ ] Tight line-height on body text (below 1.4)
- [ ] Mixed font families (3+) — one sans, one optional mono or serif
- [ ] Placeholder fonts (Arial, Georgia) anywhere visible
- [ ] Line-height above 1.15 on display headlines

### Color
- [ ] Multiple accent colors for interactive elements
- [ ] Gradient backgrounds under text
- [ ] Pure #000000 black background — use #0F0F0F minimum
- [ ] Stock blue (#003087, #0066CC, #1E40AF) as primary brand color
- [ ] Accent color appearing more than 3 times per viewport

### Layout
- [ ] Testimonial carousel / slider — replace with static grid
- [ ] Hero video background — replace with screenshot or still image
- [ ] 6-icon feature grid — cut to 3 or replace with outcome statements
- [ ] Decorative dividers (horizontal rules) — use whitespace
- [ ] Border-radius inconsistency across elements

### Copy
- [ ] Vague benefit claims ("we help advisors grow") — always quantify
- [ ] Headline that names the company before naming the problem
- [ ] "We" statements where "you" statements belong
- [ ] Generic CTAs: "Learn More," "Get Started," "Contact Us"
- [ ] Testimonials without specific outcomes
- [ ] No micro-copy under CTA buttons

### Interaction
- [ ] Missing hover states on any clickable element
- [ ] Browser-default focus rings
- [ ] All elements animating simultaneously on load — stagger by 50ms
- [ ] Mixed animation easing across the page — pick one curve
