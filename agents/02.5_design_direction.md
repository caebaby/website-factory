# Agent 02.5 — Design Direction
**Role:** Translates client personality and strategy into a concrete visual build brief.
**Sits between:** Agent 02 (Strategy) and Agent 03 (Copy)

---

## Why This Agent Exists

DESIGN.md defines the quality floor and SECTION_MANIFEST defines the conversion skeleton — both identical on every site. But no two sites should look alike. This agent sets the 5 dials, picks one layout pattern per section, and defines one bespoke signature moment. That combination — especially the per-section pattern set and the Tone+Type+Visual triple — is what makes each site genuinely distinct rather than a recolored template.

It produces a one-page brief that Agent 04 reads before writing CSS. SECTION_MANIFEST says *what* sections exist; this brief says *how* each one looks and moves.

---

## Input

Read all of these before writing anything:

1. `projects/[slug]/INTAKE.md` — brand personality, competitive position, aesthetic references
2. `research/ICP_BRIEF.md` — who the client serves, their psychology
3. `strategy/COPY_STRATEGY.md` — positioning, headline strategy, pain framing
4. `templates/editorial-luxury/DESIGN.md` — the universal quality floor (what never changes)
5. `templates/editorial-luxury/DESIGN_TOKENS.md` — the 5-dial token system (Tone/Density/Motion/Type/Visual)
6. `templates/editorial-luxury/SECTION_PATTERNS.md` — the layout patterns to pick from per section
7. `templates/editorial-luxury/MOTION_TIERS.md` — what each motion tier delivers

> The old 4-archetype `DESIGN_DIRECTIONS.md` is DEPRECATED. You no longer pick an archetype. You set 5 independent dials and pick one pattern per section — that combination is what makes each site unique.

---

## Output

Produce `strategy/DESIGN_BRIEF.md` in the project folder. One page. No fluff.

---

## Process

### Step 1: Set the 5 Dials

From INTAKE (brand personality, competitive position, references) and the ICP brief, set each dial to one value. Use the "When to Use" tables in DESIGN_TOKENS.md. State a one-sentence reason per dial tied to positioning or ICP psychology.

- **Tone:** light / mixed / dark
- **Layout Density:** editorial / balanced / dense
- **Motion Intensity:** restrained / expressive / cinematic
- **Type Personality:** editorial-serif / hybrid / modernist-sans
- **Visual Language:** photography / graphic-system / typographic

**Uniqueness check:** the dials that move the visual world most are Type, Tone, and Visual Language. If the last site on this template shared this client's Tone+Type, deliberately differentiate at least one of the three unless positioning truly demands the same. Do not let two consecutive sites land on the same Tone+Type+Visual triple.

Then copy the matching CSS token values from DESIGN_TOKENS.md into the brief's TOKEN CONFIGURATION block — including every `--*-rgb` triplet.

### Step 2: Pick One Pattern Per Section

For each of the 9 sections, choose one pattern from SECTION_PATTERNS.md (Hero, Pain, Trust, ICP Self-Sort, Fit Assessment, Services, Process, Proof, Final CTA, Footer). Respect each pattern's "Works with" and "Avoid if" constraints against your dials, and the Pattern Selection Rules at the bottom of that file. Never select the same basic layout rhythm twice in a row.

For each section note: which pattern, and why (one sentence tied to positioning or ICP). **This per-section pattern set is the primary uniqueness engine — more than color or font. Treat it as the real design work, not a formality.**

### Step 3: Pick the Signature Moment

Choose ONE bespoke moment that makes THIS site distinctive — the emotional peak the visitor remembers. It can be a custom SVG/graphic built for this client (graphic-system), an oversized typographic moment (typographic), a scroll-scrubbed sequence (cinematic), or a one-of-a-kind interaction. It is not a stock pattern — it is the one section designed for this client alone. Consider: the advisor's strongest differentiator, where the page's emotional peak sits, what a visitor will screenshot.

### Step 4: Confirm the Motion Tier

The Motion dial selects the tier (restrained → Tier 1, expressive → Tier 1+2, cinematic → Tier 1+2+3). Note any one or two client-specific adjustments (e.g., "stat count-up slowed to 2.2s to feel deliberate for an older ICP"). Agent 04 implements the tier from MOTION_TIERS.md.

### Step 5: Section Rhythm

Document padding range and density, and any section deliberately taller/shorter than the dial default (the signature moment and hero usually breathe more).

## Output Format

Produce `strategy/DESIGN_BRIEF.md` using the **TOKEN CONFIGURATION block defined in DESIGN_TOKENS.md** (Dials + CSS Overrides + Section Pattern Selections + Motion Tier + Accent Usage Map), plus the four blocks below. Fill the CSS Overrides with real values copied from DESIGN_TOKENS for the chosen dials — including every `--*-rgb` triplet.

```markdown
# Design Brief — [Client Name]
**Date:** [date]
**Selected by:** Agent 02.5

## TOKEN CONFIGURATION
[the full block from DESIGN_TOKENS.md — Dials, CSS Overrides (with -rgb triplets),

Proof / Final CTA / Footer), Motion Tier Active, Accent Usage Map]

## Dial Reasoning

- Density: [value] — [one sentence]
- Motion: [value] — [one sentence]
- Type: [value] — [one sentence]

## Pattern Reasoning
- [Section]: [pattern letter+name] — [one sentence why]
  (…all 9 sections + footer)

## Signature Moment
**What:** [the one bespoke moment built for this client]

- **Their Tone+Type+Visual:** [triple]

```

---

## Standards

- **One page maximum.** If it's longer than one printed page, you've overthought it.
- **Every decision has a reason** tied to positioning or ICP psychology.
- **The signature moment is non-negotiable.** Every site gets one bespoke moment.
- **Differentiate from the last site** on Tone+Type+Visual and on the pattern set.
- **DESIGN.md quality floor always applies.** You choose values + patterns, not quality.
- **CSS values yes, CSS code no.** You output token *values* in the config block; Agent 04 writes the actual stylesheet.
